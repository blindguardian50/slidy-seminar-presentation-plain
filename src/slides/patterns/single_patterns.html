V1: Scaling whole chart down

  • Chart is scaled down to fit parent container as is without any additional transformations being applied
  • For web-based visualizations:
    • Giving chart container relative width to parent container
  • Advantages:
    • Most straightforward and easiest solution
  • Disadvantages:
    • Components become unreadable if used too excessively
    • Uni directional scaling eventually leads to unacceptable distortions of visualization

V2: Repositioning Element Labels

  • During development visualization is inspected and labels adjusted for different visualization widths.
  • For web-based visualizations:
    • Media queries
    • Size container queries
    • CSS rotate and translate properties
  • Advantages:
    • Can solve cumbersome clutter and intersection problems
    • CSS technologies or responsive visualization tools often allow realignment of whole groups of labels
  • Disadvantages:
    • Adding too many single exceptions for repositioning can get cumbersome

V3: Using Tooltips and Hiding Element Labels

  • Labels of elements are hidden either for specific visualization widths or in general.
  • Interaction with control elements or visualization elements directly triggers tooltips becoming visible
  • For web-based visualizations:
    • CSS hover pseudo selector (works via touch interactions on mobile devices)
  • Advantages:
    • Saves space on devices which narrow screen sizes
    • Intersection and clutter of labels not possible anymore
  • Disadvantages:
    • User may overlook possibility of displaying tooltips

V4: Rotating Axis Labels

  • Rotate labels of horizontal and/or vertical axis as visualization gets narrower
  • For web-based visualizations:
    • JavaScript Event Listener and algorithm for detecting appropriate angle
    • Application of rotation via CSS rotate property
  • Advantages:
    • Prevents clutter and intersection of horizontal axis labels
    • Preserves original information of horizontal axis labels
    • Rotation of vertical axis labels saves space in horizontal direction
  • Disadvantages:
    • Readability of axis labels decreases as natural reading direction is not considered

V5: Reformatting Titles and Labels

  • Use well-known formats to shorten labels and titles
  • Well-known formats:
    • Numbers: 2.000 = 2k, 2.000.000 = 2M, etc.
    • Dates: January 1st 2024 = 01-01-2024 ~ 01-2024 ~ 01-24
    • Geographic locations: United States of America = USA
    • Organization names: Web Consortium Cascading Style Sheets Working Group = W3C CSS WG
  • Reformulate titles if no format exists
  • For web-based visualizations:
    • Responsive visualization tools often provide configuration options for formatting labels
  • Advantages:
    • Prevents clutter and intersections
  • Disadvantages:
    • Readability of labels highly dependent on used format and user
    • Shortening of labels can lead to information loss

V6: Scaling Labels between minimal and maximal Tolerances

  • Adapting font size of labels either fluently or via breakpoints
  • For web-based visualizations:
    • CSS media queries for changing font size at breakpoints
    • CSS size container queries for changing font size at breakpoints
    • CSS comparison functions for fluidly changing font size
    • CSS comparison functions can be combined with container query units to achieve better control
  • Advantages:
    • Overall aesthetics of visualization improve if font size is in relation to visualization size
    • Reduces risk of clutter and intersections
    • Tolerance values prevent extraordinary small or large font sizes
  • Disadvantages:
    • Fluid Transition between labels not exactly predictable and requires manual testing

V7: Scaling down selection of elements

  • Approach varies for different kinds of elements:
    • Bar elements: Can be scaled without problems in horizontal and vertical direction
    • Line elements: Can be scaled without problems in horizontal and vertical direction
    • Point elements (or other markers): Must keep aspect-ratio to maintain their shape
  • Approach varies for different kinds of visualizations:
    • Visualizations with variable aspect-ratio (cartesian charts, parallel coordinates chart, etc.)
    • Visualizations with fixed aspect-ratio (pie chart, chord diagram, maps, etc.)
  • For web-based visualizations:
    • JavaScript algorithm for recalculating new position and size of elements
  • Advantages:
    • Can save a lot of space on narrower screens in both horizontal and vertical direction
    • Smooth transition if implemented via event listener and algorithm, which appears very natural
  • Disadvantages:
    • Elements must already be small on wider spaces if a large number of elements is given
    • Visualizations with fixed aspect-ratio require additional considerations
    • Elements with fixed aspect-ratio require additional considerations
    • Line elements appear steep on narrow screens and flat on wider ones

V8: Removing categories, dimensions, elements and axis labels

  • Removing information from a visualization as space gets narrower
  • For web-based visualizations:
    • Provide less data to visualization tools as screen gets narrower
    • Hide elements, categories or dimensions via CSS container and media queries
  • Advantages:
    • Can save a lot of space on narrower screens in both horizontal and vertical direction
  • Disadvantages:
    • Information loss must be taken into account
    • The concealment of essential information can change the original message of the visualization

V9: Swapping Data Dimensions

  • Different data dimensions often have different scales and require a different amount of space
  • Swapping data dimensions can exploit this aspect
  • Especially swapping of cartesian chart axis is a useful version of this pattern
  • Advantages:
    • Can transform visualization to better fit current space requirements without loosing information
  • Disadvantages:
    • Major change in visualization which may affect other ongoing transformations if used incautiously

V10: Swapping Whole Visualization

  • Sometimes visualizations cannot be transformed because transformation would result in:
    • Unacceptable amount of information loss
    • Alteration of original message
    • Distortion of visualization
    • Unreadable elements or labels
  • As other transformations lead to unacceptable results, visualization is completely swapped with different visualization that maintains the original message as well as possible
  • Advantages:
    • Provides a solution where all other transformations do not
  • Disadvantages:
    • Creating two visualizations requires more effort than creating a single one
    • It may be difficult to preserve the original message for certain chart types

I1: Adding Toolbar and Menus

  • Provide interactions via additional control elements
  • Can be hidden on default and revealed on first interaction with chart to save space
  • Highlighting whole visualization and switching cursor into a pointer on hover increases chance of being discovered
  • Typical functionalities provided by toolbars and menus:
    • Downloading a visualization
    • Show/Hide categories
    • Show/Hide dimensions
    • Selection between different kinds of views (maps)
    • Selection between different interaction types (zoom)
  • Advantages:
    • Provides theoretically endless options for additional interactions
    • Likely to be discovered by the user
  • Disadvantages:
    • Takes up space if not hidden by default
    • Must be discovered by user if hidden by default

I2: Limiting Selection of visible Categories, Records and Dimensions

  • Include control elements enabling to show/hide categories, dimensions and/or records
  • Prevent categories, dimensions and/or records from being visible at the same time
  • Advantages:
    • Hidden data can be made visible again by the user
  • Disadvantages:
    • Not all information of a visualization is visible at the same time

I3: Add Zooming

  • Enabling user to zoom in and out of visualization
  • Enabling user to pan when zoomed into part of visualization
  • Optionally include control elements for switching between different types of zoom
  • Advantages:
    • Gives user possibility to look into especially interesting parts of a visualization
    • Makes otherwise inaccessible parts of a visualization accessible
  • Disadvantages:
    • Not all information of a visualization is visible at the same time

D1: Applying Data Generalization

  • Form of data mining and typically used to abstract personal data
  • Classify many data points into several managable groups
  • Allows presenting information in a more compact way

D2: Applying Data Aggregation

  • Summarize information data set into one or multiple useful statistics
  • Allows presentation of theoretically arbitrary large data collections as single visualization element
  • Can be used well in combination with data generalization to represent multiple semantically related data points as single visualization element

D3: Applying Cluster Analysis

  • Method of unsupervised learning in field of data analysis
  • Group unlabeled objects based on their similarities
  • Common types of clustering:
    • Hierarchical clustering
    • K-means clustering
    • Model-based clustering
    • Density-based clustering
    • Fuzzy clustering
  • Allows abstraction of similar data points in data set
  • Example 1: Usage of automatic hierarchical clustering to replace intersecting data points with clusters
  • Example 2: Provide control elements for applying clustering on demand
